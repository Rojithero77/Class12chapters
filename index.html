Chapter 1: Database Fundamentals

1. Data

Data refers to raw facts or figures without context. For example, numbers, names, and dates.

2. Information

Information is processed data that has meaning and value. For example, a report showing student scores.

3. Features of Information

  • Accuracy
  • Relevance
  • Completeness
  • Timeliness

4. Database and its Purpose

A database is an organized collection of data. Its purpose is to store, manage, and retrieve data efficiently.

5. Terminologies in Database

  • Table: A set of data elements organized in rows and columns.
  • Field: A single piece of data; a column in a table.
  • Record/Tuple: A row in a table.
  • Object: A combination of data and its behavior.
  • Keys: Unique identifiers for records (e.g., Primary Key).

6. Data Dictionary

It is a set of information describing the contents, format, and structure of a database and the relationship between its elements.

7. Database Management System (DBMS)

Introduction

A software that interacts with users, applications, and the database itself to capture and analyze data.

Objectives

  • Data integrity
  • Data security
  • Concurrent access

Advantages

  • Efficient data management
  • Data sharing and backup

Disadvantages

  • High cost
  • Complexity

8. Types of Database Model

  • Hierarchical: Data is organized in a tree-like structure.
  • Network: More flexible than hierarchical, supports many-to-many relations.
  • Relational: Data organized in tables.
  • Entity-Relational: Uses ER diagrams to model data.

9. Integrity Constraints and Types

  • Domain: Ensures data falls within a specific range.
  • Entity: Ensures each row has a unique identifier.
  • Referential: Ensures consistency among rows in different tables.

10. Keys

Keys are used to uniquely identify rows in a table. Examples include Primary Key, Foreign Key, Candidate Key.

11. Normalization

Introduction

Process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF: Eliminate repeating groups
  • 2NF: Remove partial dependency
  • 3NF: Remove transitive dependency

Advantages

  • Reduces data redundancy
  • Improves data consistency

Disadvantages

  • Complex queries
  • Performance overhead

12. Centralized and Distributed Database

Introduction

Centralized databases are stored in one location; distributed are spread across multiple locations.

Advantages

  • Centralized: Easier to manage
  • Distributed: Improved performance and reliability

Disadvantages

  • Centralized: Risk of failure
  • Distributed: Complex to implement

Comparison

Each has its pros and cons; choice depends on application needs.

13. Database Security

Introduction

Protecting data from unauthorized access and threats.

Challenges

  • SQL injection
  • Data breaches

Security Measures

  • User authentication
  • Access control

Roles of DBA

  • Install and configure DBMS
  • Monitor performance
  • Ensure security

14. Practical Topics

DDL and DML Languages

DDL (Data Definition Language) defines database structures. DML (Data Manipulation Language) manipulates data.

SQL Data Types

  • CHAR, VARCHAR, TEXT: Store character data
  • BINARY, VARBINARY, BLOB: Store binary data
  • ENUM, BIT, BOOLEAN: Store logical values
  • INT, FLOAT, DOUBLE, DECIMAL: Numeric types
  • DATE, DATETIME: Temporal data types
